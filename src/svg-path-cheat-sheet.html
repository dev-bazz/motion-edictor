Path Draw-On (stroke-dashoffset)

Basic Draw-On Core Technique
CSS 
/* 1. Measure path length in JS or DevTools */
/* path.getTotalLength() → e.g. 300 */

.path {
  stroke-dasharray: 300;   /* total length */
  stroke-dashoffset: 300; /* start hidden */
  animation: draw 2s ease forwards;
}

@keyframes draw {
  to { stroke-dashoffset: 0; }
}
Dynamic Length via CSS Var Pro Tip
CSS + JS 
/* Set length dynamically from JS */
const path = document.querySelector('path');
path.style.setProperty(
  '--len', path.getTotalLength()
);

.path {
  stroke-dasharray:  var(--len);
  stroke-dashoffset: var(--len);
  animation: draw 2s ease forwards;
}

@keyframes draw {
  to { stroke-dashoffset: 0; }
}
Erase Animation (Reverse) Technique
CSS 
/* Animate from 0 → length to erase */
.path-erase {
  stroke-dasharray:  300;
  stroke-dashoffset: 0; /* start visible */
  animation: erase 2s ease forwards;
}

@keyframes erase {
  to {
    /* negative = erase from end */
    stroke-dashoffset: -300;
  }
}
Scroll-Driven Draw Modern CSS
↓ scroll to animate in real usage
CSS (Chrome 115+) 
.path {
  stroke-dasharray: 400;
  stroke-dashoffset: 400;
  animation: draw linear;
  animation-timeline: scroll();
  animation-range: entry 0% cover 60%;
}

@keyframes draw {
  to { stroke-dashoffset: 0; }
}
ℹ️
How it works stroke-dasharray creates a dashed pattern. Setting it to the path length creates one "dash" the full length of the path. stroke-dashoffset shifts it — starting at length hides it completely, animating to 0 draws it. Negative offset erases from the end.

Marching Ants

CSS

  stroke-dasharray: 20 8;

Motion Path (offset-path)

Translate Along a Path CSS Motion Path
CSS
.ball {
  offset-path:
    path("M10,80 C40,10 80,150
       120,80 S180,10 220,80");
  offset-rotate: auto; /* face dir */
  offset-anchor: center;
  animation: travel
    3s linear infinite;
}
@keyframes travel {
  from { offset-distance: 0%; }
  to   { offset-distance: 100%; }
}
Orbit (circular path) offset-rotate: 0deg
CSS
/* SVG arc path for circle */
/* M cx,top a r,r 0 1 1 0,2r ... */
.planet {
  offset-path: path(
    "M80,20 a60,60 0 1,1
     0,120 a60,60 0 1,1 0,-120"
  );
  offset-rotate: 0deg; /* no flip */
  animation: orbit
    3s linear infinite;
}
@keyframes orbit {
  to { offset-distance: 100%; }
}
Bounce Along Curve easing magic
CSS
.ball {
  offset-path:
    path("M20,100 Q100,20 220,100");
  offset-rotate: auto;
  animation: bounce
    2s cubic-bezier(.36,.07,.19,.97)
    infinite;
}
@keyframes bounce {
  0%,100%{offset-distance:0%}
  50%    {offset-distance:100%}
}
offset-rotate Reference All Values
auto Rotate to face travel direction
auto 90deg auto + 90° offset
0deg Keep original orientation
reverse Face backwards

Path Morphing

CSS d property morph Modern CSS
CSS (Chrome 93+)
/* CRITICAL: same # of points! */
.shape {
  d: path("M40,80 C40,40
     80,40 80,80 S120,120 120,80");
  animation: morph
    3s ease-in-out infinite;
}
@keyframes morph {
  50% {
    d: path("M40,80 C80,20
       80,140 120,80 S160,40 160,80");
  }
}
SMIL <animate> Morph All browsers
SVG/SMIL
<!-- Works in all modern browsers -->
<path fill="none">
  <animate
    attributeName="d"
    values="M30,60 C...;
            M30,60 C...;"
    dur="3s"
    repeatCount="indefinite"
  />
</path>
Morph Requirements & Tricks

✓ Requirements

  • 🔢 Same number of path commands
  • 🔤 Same command types (M, C, L...)
  • 📍 Same command order
  • 🔗 Same start/end point type

✦ Tricks

  • 🔄 Convert all to cubic bezier (C)
  • 📐 Add dummy points to match count
  • 🛠 Use tools: Flubber.js, GSAP MorphSVG
  • 🔁 Interpolate via Web Animations API

Staggered Path Animation

CSS animation-delay Stagger nth-child
CSS
.group path {
  stroke-dasharray: 200;
  stroke-dashoffset: 200;
  animation: draw
    1.5s ease forwards;
}

.group path:nth-child(1) {
  animation-delay: 0s;
}
.group path:nth-child(2) {
  animation-delay: 0.2s;
}
/* Or generate with JS/SCSS loop */
JS Dynamic Stagger Loop JS
JavaScript
const paths =
  document.querySelectorAll('path');

paths.forEach((path, i) => {
  const len = path.getTotalLength();
  path.style.setProperty(
    '--len', len
  );
  path.style.animationDelay =
    `${i * 0.15}s`;
});

Complete Properties Reference

Stroke & Path CSS Properties
Property Values Notes
stroke-dasharray none | <length>+ Defines dash and gap pattern. Single value = equal dash/gap. Multiple values cycle.
stroke-dashoffset <length> | <percentage> Shifts dash pattern along path. Negative shifts backwards. Animatable.
stroke-linecap butt | round | square End cap shape. round for smooth animated lines.
stroke-linejoin miter | round | bevel Corner join style at path segments.
stroke-width <length> Animatable. Use for pulse effects.
stroke-opacity 0 – 1 Separate from opacity. Composited differently with fill.
fill-opacity 0 – 1 Animate for fade-in effects on filled shapes.
vector-effect non-scaling-stroke Keeps stroke width constant regardless of scale transform.
d path("...") CSS property for path shape. Animatable for morph (Chrome 93+).
offset-path path() | url() | ray() | basic-shape Defines the motion path for the element.
offset-distance 0% – 100% Position along the offset-path. Animate this for motion.
offset-rotate auto | auto <angle> | <angle> Orientation of element along path. auto = faces travel direction.
offset-anchor <position> The element's anchor point on the path. Default center.
pathLength <number> SVG attribute! Normalizes path length. Set to 1 then use 0–1 for dasharray.
✨
pathLength="1" trick Set pathLength="1" on any SVG path, then use stroke-dasharray: 1 and stroke-dashoffset: 1. No need to call getTotalLength() — it normalizes the length to 1. Works everywhere.

JavaScript APIs

SVG Path DOM Methods
JavaScript
const path = document.querySelector('path');

// Total path length in px
const len = path.getTotalLength();

// Point at position (0–len)
const pt = path.getPointAtLength(len * 0.5);
// → { x: 120.3, y: 45.8 }

// Use for placing elements along path
dot.style.transform =
  `translate(${pt.x}px, ${pt.y}px)`;

// Manual progress animation
let progress = 0;
function tick() {
  progress = (progress + 1) % len;
  const p = path.getPointAtLength(progress);
  dot.style.transform =
    `translate(${p.x}px, ${p.y}px)`;
  requestAnimationFrame(tick);
}
tick();
Web Animations API
JavaScript
const path = document.querySelector('path');
const len  = path.getTotalLength();

// WAAPI — programmatic control
const anim = path.animate(
  [
    { strokeDashoffset: len },
    { strokeDashoffset: 0   }
  ],
  {
    duration: 2000,
    easing: 'cubic-bezier(0.4,0,0.2,1)',
    fill: 'forwards'
  }
);

// Control playback
anim.pause();
anim.play();
anim.reverse();
anim.currentTime = 500; // seek
anim.playbackRate = 2;  // 2× speed

// Promises
await anim.finished;
Trigger on Scroll (IntersectionObserver)
JavaScript
const paths = document.querySelectorAll('.animate-path');

const io = new IntersectionObserver(
  (entries) => {
    entries.forEach(entry => {
      if (entry.isIntersecting) {
        entry.target.classList.add('play');
        io.unobserve(entry.target);
      }
    });
  },
  { threshold: 0.2 } // 20% visible
);

paths.forEach(p => io.observe(p));

/* CSS: .animate-path { animation-play-state: paused; }
        .animate-path.play { animation-play-state: running; } */

Gotchas, Fixes & Performance

Common Bugs
🐛
Stroke not animating Forgot to set both stroke-dasharray AND stroke-dashoffset to the same value. Both are required to hide the path initially.
🐛
transform-origin broken on SVG SVG transforms use the SVG coordinate system, not the element center. Use transform-box: fill-box + transform-origin: center to fix.
🐛
offset-path jumps on loop Object teleports back to start. Use animation-direction: alternate or set start keyframe explicitly in your @keyframes.
🐛
Path morph fails silently Source and target paths must have the same number and type of commands. Use absolute commands (uppercase letters) only.
Performance Tips
⚡
Use will-change will-change: stroke-dashoffset promotes the element to its own compositor layer — eliminates jank on complex paths.
⚡
Animate only composited properties stroke-dashoffset, opacity, and transform are GPU-composited. Avoid animating stroke-width or fill in tight loops.
⚡
Reduce path complexity Simplify paths with tools like SVGOMG. Fewer control points = faster rendering.
⚡
prefers-reduced-motion Always wrap in @media (prefers-reduced-motion: reduce) to disable or minimize for accessibility.
Quick-win Snippets
CSS Utilities
/* ① Fix transform-origin in SVG */
.svg-element {
  transform-box: fill-box;
  transform-origin: center;
}

/* ② pathLength normalization trick */
<path pathLength="1" />  /* SVG attr */
.path { stroke-dasharray: 1; stroke-dashoffset: 1; }

/* ③ Accessibility */
@media (prefers-reduced-motion: reduce) {
  * { animation-duration: 0.01ms !important; }
}

/* ④ GPU layer promotion */
.animated-path {
  will-change: stroke-dashoffset;
}

/* ⑤ Pause/play with class */
.path            { animation-play-state: paused; }
.path.is-playing { animation-play-state: running; }

/* ⑥ Consistent direction — always use absolute path commands */
/* M (absolute) vs m (relative) — stick to UPPERCASE */

/* ⑦ non-scaling-stroke for zoom-safe SVGs */
path { vector-effect: non-scaling-stroke; }
