Container Queries

Baseline 2023
Basic Syntax 
/* 1. Establish a container */
.parent {
  container-type: inline-size;
  container-name: card;
  /* shorthand: */
  container: card / inline-size;
}

/* 2. Query from a descendant */
@container card (width > 400px) {
  .child { font-size: 1.5rem; }
}

container-type Values 
/* Responds to inline axis (width) only */
container-type: inline-size;

/* Responds to BOTH width and height */
container-type: size;

/* No containment — default */
container-type: normal;

/* Named shorthand */
container: hero / size;
container: card / inline-size;

/* ⚠ 'size' needs explicit height on container */
NAMED CONTAINERS — sidebar vs main
Range Syntax & Container Units 
/* Modern range syntax (preferred over min-/max-) */
@container (200px < width < 600px) {}
@container (width >= 400px) {}

/* Container length units */
cqw   /* 1% of container width  */
cqh   /* 1% of container height */
cqi   /* 1% of inline size      */
cqb   /* 1% of block size       */
cqmin /* min(cqi, cqb)          */
cqmax /* max(cqi, cqb)          */

/* Fluid type with clamp + cqi */
.title {
  font-size: clamp(1rem, 5cqi, 3rem);
}

Aspect-ratio & Boolean Logic 
/* Query aspect-ratio (requires size type) */
.box { container-type: size; }

@container (aspect-ratio > 1) {
  /* wider than tall */
}

/* Logical dimensions */
@container (inline-size > 400px) {}
@container (block-size > 300px) {}

/* Boolean AND / OR / NOT */
@container (width > 300px) and (height > 200px) {}
@container (width > 300px) or (width < 100px) {}
@container not (width > 600px) {}

Style Queries

Baseline 2024
Custom Property Queries 
/* No container-type required for style() */
.wrapper { --theme: dark; }

@container style(--theme: dark) {
  .child {
    background: #0a0a0f;
    color: white;
  }
}

/* Query multiple values */
@container style(--size: large) {
  .child { padding: 2rem; }
}

Named Style Containers 
/* Combine name + style query */
.wrapper {
  container-name: theme-ctx;
  --variant: primary;
}

@container theme-ctx style(--variant: primary) {
  .btn { background: blue; }
}

@container theme-ctx style(--variant: danger) {
  .btn { background: red; }
}

/* Works across any depth in subtree */

Boolean & Combined Checks 
/* Boolean: truthy if prop != initial value */
@container style(--active) {}

/* NOT */
@container style(not (--dark-mode: true)) {}

/* AND — both conditions must match */
@container style(
  (--theme: dark) and (--size: large)
) {}

/* OR — either condition matches */
@container style(
  (--type: a) or (--type: b)
) {}

/* Nesting logic */
@container style((--a) and (not (--b))) {}

Inherited Standard Properties 
/* Only INHERITED properties are queryable */

/* ✓ These work: */
@container style(font-style: italic) {
  em { font-style: normal; }
}
/* ✓ Also: color, font-size, direction,
         writing-mode, visibility        */

/* ✗ These do NOT work: */
/* width, display, position, flex, grid  */

/* Best practice: use --custom-props */
:root { --surface: light; }
[data-theme="dark"] { --surface: dark; }

@container style(--surface: dark) {
  .card { color: white; }
}

Scroll-Driven Animations

Baseline 2024
scroll() — Progress Timeline 
/* Tie animation to scroll progress 0–100% */
.bar {
  animation-timeline: scroll(root block);
  animation: grow linear both;
}

/* scroll( scroller  axis ) */
/* scroller: root | nearest | self   */
/* axis:     block | inline | x | y  */

@keyframes grow {
  from { transform: scaleX(0); }
  to   { transform: scaleX(1); }
}

/* animation-fill-mode: both is critical! */
SCROLL INSIDE BOX — watch progress bar
animation-timeline: scroll(nearest block)
bar fills as you scroll — no JS
GPU-accelerated via compositor
works with nested scrollers
scroll() tracks 0% → 100% progress
combine with animation-range to clip
keep scrolling ↓
view() — Visibility Timeline 
/* Animate as element enters the viewport */
.card {
  animation-timeline: view(block);
  animation-range: entry 0% entry 60%;
  animation: appear linear both;
}

/* animation-range keywords:
   entry          exit
   contain        entry-crossing
   exit-crossing                  */

@keyframes appear {
  from { opacity: 0; scale: 0.95; }
  to   { opacity: 1; scale: 1;    }
}
SCROLL TO REVEAL items
▸ entry 0% → element begins entering scroller
▸ entry 60% → animation completes
▸ fades in as element enters visible area
▸ no IntersectionObserver needed
▸ pure CSS, GPU composited
▸ combine: scale + blur + opacity
Named Scroll & View Timelines 
/* Named scroll timeline on the scroller */
.scroller {
  scroll-timeline: --page-scroll block;
  /* or split properties: */
  scroll-timeline-name: --page-scroll;
  scroll-timeline-axis: block;
}

/* Named view timeline on the subject */
.hero {
  view-timeline: --hero-tl block;
}

/* Reference by name from ANYWHERE in subtree */
.parallax-el {
  animation: drift linear;
  animation-timeline: --hero-tl;
}
SCROLL — view timeline items animate in
timeline name: --named-tl
defined on the scroll container
referenced by name from any child
decoupled from DOM hierarchy
multiple named timelines per page
axis: block | inline | x | y

Gotchas & Footguns

Can't query own container
A container cannot query itself — only descendants can.

✗ @container (…) { .parent {} }
✓ @container (…) { .child {} }
size type needs explicit height
container-type: size requires a definite block size. Without an explicit height on the container, block-axis queries never fire.

✓ Set height, block-size, or constrain via grid/flex parent.
Style queries: custom props only
Only --custom-props work reliably in style() today. Querying standard inherited props is experimental — don't ship in prod yet.

✓ Use custom props as an explicit theming API.
cq units fall back to vw/vh
If no size container exists in the ancestor chain, cqw/cqi fall back to the viewport — causing unexpected full-width layouts.

✓ Always wrap cq-unit users in a size container.
scroll() needs overflow: auto/scroll
scroll(nearest) walks up the DOM for a scrolling ancestor. If it finds none (e.g. overflow: hidden), it falls through to the viewport.

✓ Use scroll(root) for page-level progress bars.
animation-fill-mode: both
Scroll animations default to fill: none. Without both, elements snap back to their from state when the range exits.

✓ Always add animation-fill-mode: both to scroll-driven animations.
Reduced motion accessibility
Scroll-driven animations are motion. Always guard:

@media (prefers-reduced-motion: reduce) {
.animated { animation: none; }
}
Named timelines are scoped
Named scroll/view timelines are only reachable by descendants of the defining element — not siblings or parents. Plan your DOM accordingly.
