What is Scroll-Driven Animation? Core Concept

Instead of time progressing the animation, scroll position does. Two core timeline types power everything:

📜
ScrollTimeline
scroll()
Tracks position within a scroll container. 0% = top, 100% = bottom. Use for progress bars, parallax.
👁
ViewTimeline
view()
Tracks element visibility inside scrollport. Use for reveal-on-scroll, entrance/exit effects.
⚡ Performance Runs on the compositor thread — buttery 60fps+ with zero main-thread jank. Far superior to JS scroll listeners.

view-timeline Property

CSS/* Declare on the observed element */ .card { view-timeline-name: --card-reveal; view-timeline-axis: block; /* Optional: shrink the trigger area */ view-timeline-inset: 100px 0px; } /* Shorthand */ .card { view-timeline: --card-reveal block; }

animation-timeline Values

Value Description
auto Default time-based timeline
none No timeline — animation frozen
scroll() Anonymous scroll progress timeline
view() Anonymous view progress timeline
--my-timeline Named timeline reference (dashed-ident)
CSS@keyframes fade-in { from { opacity: 0; transform: translateY(40px); } to { opacity: 1; transform: translateY(0); } } .element { animation: fade-in linear both; animation-timeline: scroll(); /* ← magic */ }
ℹ Note animation-duration is ignored when using scroll-driven timelines — scroll position controls progress, not time.

scroll() — Anonymous Scroll Timeline Function

Syntaxanimation-timeline: scroll(<scroller> <axis>); /* <scroller> */ nearest /* default — nearest scrolling ancestor */ root /* document viewport */ self /* the element itself */ /* <axis> */ block /* default — vertical (in LTR) */ inline /* horizontal (in LTR) */ x /* explicit horizontal */ y /* explicit vertical */
CSS/* Examples */ animation-timeline: scroll(); /* nearest, block */ animation-timeline: scroll(root); /* page scroll */ animation-timeline: scroll(root block); /* explicit */ animation-timeline: scroll(self inline); /* horizontal self */ animation-timeline: scroll(nearest y); /* vertical ancestor */

view() — Anonymous View Timeline Function

Syntaxanimation-timeline: view(<axis> <inset>); /* <inset> — shrinks the trigger zone */ auto /* default */ 100px /* both start and end */ 20% /* percentage */ 100px 50px /* start-inset end-inset */
CSS/* Basic — full visibility range */ animation-timeline: view(); /* With 100px inset on both sides */ animation-timeline: view(block 100px); /* Horizontal scrolling gallery */ animation-timeline: view(inline); /* Asymmetric inset */ animation-timeline: view(block 80px 20px);
ℹ Inset Explained view(block 100px) shrinks the scrollport's trigger area by 100px on each side. The animation only progresses when the element is within that inner zone.

animation-range Core

Controls which slice of the timeline the animation maps to. Without this, 0%–100% of the full timeline is used.

cover
Enters → fully exits scrollport. Full lifecycle.
contain
While fully visible inside scrollport.
entry
Starts entering → fully entered.
exit
Starts exiting → fully exited.
entry-crossing
Crosses the entry edge of scrollport.
exit-crossing
Crosses the exit edge of scrollport.

animation-range Examples

CSS/* Only during entry phase */ .card { animation-range: entry 0% entry 100%; } /* Percentage of scroll distance */ .progress { animation-range: 25% 75%; } /* Pixel offset */ .header { animation-range: 0px 200px; } /* Mixed: entry start to exit start */ .element { animation-range: entry 0% exit 0%; }

animation-fill-mode with Ranges

Fill Mode Before Range After Range
none No styles No styles
forwards No styles Final keyframe
backwards First keyframe No styles
both First keyframe Final keyframe
💡 Always use both animation-fill-mode: both prevents flash-of-unstyled-element before entering and after exiting the animation range.

⑩ Image Sequence on Scroll Recipe

Step through a sprite sheet frame-by-frame using steps() easing — the Apple product reveal technique.

CSS@keyframes sprite { from { background-position: 0% 0; } to { background-position: 100% 0; } } .sprite-player { width: 200px; height: 200px; /* Sheet is 10 frames wide — so 1000% wide */ background: url('spritesheet.png') 0 0 / 1000% 100% no-repeat; /* steps(9) = 10 frames, 9 steps between */ animation: sprite steps(9, end) both; animation-timeline: scroll(root); }
💡 Apple Style This powers Apple's cinematic product pages — hundreds of PNGs, stepped through by scroll position to create a 3D rotation or exploded-view effect. The key is steps(N-1, end) where N is your frame count.

Named Timelines Advanced

Define a timeline on a container, reference it from any descendant.

CSS/* Container defines the timeline */ .scroll-container { overflow-y: scroll; scroll-timeline: --main-scroll block; } /* Multiple descendants can all reference it */ .progress-bar { animation: grow linear both; animation-timeline: --main-scroll; } .sidebar-dot { animation: move-dot linear both; animation-timeline: --main-scroll; } .parallax-bg { animation: shift linear both; animation-timeline: --main-scroll; }

timeline-scope Advanced

Hoists a named timeline to an ancestor so sibling elements can cross-reference it.

CSS/* Without timeline-scope: siblings can't share timelines */ /* With it: hoist to common ancestor */ .layout { timeline-scope: --card-view; /* ancestor claims it */ } .layout .card { view-timeline: --card-view block; /* child A defines */ } .layout .sidebar-marker { animation: highlight linear both; animation-timeline: --card-view; /* sibling B uses it */ }
⚠️ Rule timeline-scope must be on a common ancestor of both the timeline source and consumer. It doesn't create timelines — it broadcasts them upward.

Mixing Scroll + Time-based Animations Pattern

CSS@keyframes scroll-fade { from { opacity: 0; } to { opacity: 1; } } @keyframes pulse { 0%, 100% { scale: 1; } 50% { scale: 1.05; } } .card { /* Comma-separated: each animation gets its own timeline */ animation: scroll-fade linear both, pulse 2s ease-in-out infinite; animation-timeline: view(), /* scroll-fade — view-driven */ auto; /* pulse — time-driven */ }

Timeline Range Names

Name Description Use Case
