📜 CSS Scroll Animation

Comprehensive cheat sheet — properties, timelines, ranges, recipes & live previews

Overview & Concepts

CSS Scroll-Driven Animations tie @keyframes progress to scroll position instead of time. Two core timeline types:

Timeline Type What Drives It Use Case
ScrollTimeline Position of scroll container Progress bars, parallax, sticky headers
ViewTimeline Element visibility in scrollport Reveal-on-scroll, entrance/exit animations
💡 Key Insight CSS scroll-driven animations run on the compositor thread — buttery-smooth 60fps+ with zero main-thread jank, unlike JS scroll listeners.

scroll-timeline Property

Declares a named scroll progress timeline on a scroll container.

Property Values Default
scroll-timeline-name none | <dashed-ident> none
scroll-timeline-axis block | inline | x | y block
scroll-timeline Shorthand: <name> <axis>
CSS/* Name a scroll timeline on a container */ .scroll-container { overflow-y: scroll; scroll-timeline-name: --page-scroll; scroll-timeline-axis: block; } /* Shorthand */ .scroll-container { overflow-y: scroll; scroll-timeline: --page-scroll block; }

view-timeline Property

Declares a view progress timeline — measures how much of an element is visible in the scrollport.

Property Values Default
view-timeline-name none | <dashed-ident> none
view-timeline-axis block | inline | x | y block
view-timeline-inset auto | <length-percentage>{1,2} auto
view-timeline Shorthand: <name> <axis>
CSS.card { view-timeline-name: --card-reveal; view-timeline-axis: block; } /* Shorthand */ .card { view-timeline: --card-reveal block; }

animation-timeline Core

The single most important property — links an animation to a scroll or view timeline.

Value Description
auto Default time-based timeline
none No timeline — animation won't progress
scroll() Anonymous scroll progress timeline
view() Anonymous view progress timeline
<dashed-ident> Named timeline reference
CSS@keyframes fade-in { from { opacity: 0; transform: translateY(40px); } to { opacity: 1; transform: translateY(0); } } .element { animation: fade-in linear both; animation-timeline: scroll(); }
ℹ️ Note When using animation-timeline, animation-duration is effectively ignored — progress comes from scroll position, not time.

scroll() Function Anonymous

Creates an anonymous scroll progress timeline inline.

Syntaxanimation-timeline: scroll(<scroller> <axis>); /* <scroller> values */ nearest /* default — nearest scroll ancestor */ root /* document viewport */ self /* the element itself */ /* <axis> values */ block /* default — vertical in LTR */ inline /* horizontal in LTR */ x /* explicit horizontal */ y /* explicit vertical */
CSS/* Progress bar tied to root scroll */ .progress-bar { animation: grow-width linear both; animation-timeline: scroll(root block); } /* Element scrolls itself */ .self-scroll { overflow-y: scroll; animation: highlight linear both; animation-timeline: scroll(self); }

view() Function Anonymous

Creates an anonymous view progress timeline — tracks element visibility in scrollport.

Syntaxanimation-timeline: view(<axis> <inset>); /* <inset> — shrink scrollport trigger area */ auto /* default */ 100px /* both start and end */ 20% /* percentage */ 100px 50px /* start-inset end-inset */
CSS/* Fade in when element enters viewport */ .card { animation: fade-in linear both; animation-timeline: view(); } /* With inset — trigger 100px before edge */ .card { animation: fade-in linear both; animation-timeline: view(block 100px); }
ℹ️ Insets Explained view(block 100px) virtually shrinks the scrollport by 100px on each side. The element must scroll 100px past the edge before the animation begins.

Property Values

animation-range-end normal | <length-%> | <range-name> <%>
animation-range Shorthand: <start> <end>

animation-fill-mode Behavior

Fill Mode Before Range Start After Range End
none No applied styles No applied styles
forwards No applied styles Final keyframe
backwards First keyframe No applied styles
both First keyframe Final keyframe
💡 Tip Always use animation-fill-mode: both for scroll-driven animations so the element holds its state before entering and after exiting the range.

Named Timelines Advanced

Reference a specific scroller or element for cross-element animations.

CSS/* Container defines the named scroll timeline */ .scroll-container { overflow-y: scroll; scroll-timeline: --main-scroll block; } /* Any descendant can reference it */ .progress-bar { animation: grow linear both; animation-timeline: --main-scroll; } .sidebar-indicator { animation: move-dot linear both; animation-timeline: --main-scroll; }

timeline-scope Advanced

timeline-scope on an ancestor "hoists" a named timeline so sibling elements can reference it.

CSS/* The common ancestor hoists the scope */ .layout { timeline-scope: --card-timeline; } /* Child A defines the timeline */ .layout .card { view-timeline: --card-timeline block; } /* Child B (sibling) uses the timeline */ .layout .sidebar-marker { animation: slide linear both; animation-timeline: --card-timeline; }
⚠️ Important timeline-scope must be on a common ancestor of both the timeline source and consumer. It doesn't create a timeline — it widens accessibility.

Multiple Animations, Different Timelines Pattern

CSS@keyframes scroll-fade { from { opacity: 0; } to { opacity: 1; } } @keyframes pulse { 0%, 100% { scale: 1; } 50% { scale: 1.05; } } .card { /* First: scroll-driven, Second: time-based */ animation: scroll-fade linear both, pulse 2s ease-in-out infinite; animation-timeline: view(), /* scroll-fade */ auto; /* pulse (time-based) */ }

Best Practices Guide

# Guideline Why
1 Use transform & opacity only Compositor-friendly — smooth 60fps
2 Always animation-fill-mode: both No flashing before/after range
3 @supports for detection Graceful degradation
4 Prefer view() for per-element Simpler than named timelines
5 scroll(root) for page-level Explicit scroller reference
6 will-change: transform sparingly Saves GPU memory
7 Use linear easing User controls speed via scroll
8 Add polyfill for production Safari/Firefox still limited
9 Avoid height/width/top/left Trigger layout recalc
10 Test with DevTools Animations panel Visual scroll-timeline debugging
💡 Performance CSS scroll-driven animations are significantly faster than JS scroll listeners — they run off the main thread. Always prefer them over requestAnimationFrame scroll handlers.
