1. Overview & Concepts

CSS Scroll-Driven Animations let you tie @keyframes progress to scroll position instead of time. There are two core timeline types:

Timeline Type What drives it Use Case
ScrollTimeline Position of scroll container Progress bars, parallax, sticky animations
ViewTimeline Element's visibility in scroll-port Reveal-on-scroll, entrance/exit animations

The animation's current time is determined by scroll progress (0% → 100%) rather than elapsed seconds. All standard @keyframes rules apply — you just swap the time-based timeline for a scroll-based one.

💡 Key Insight

Unlike JavaScript scroll listeners, CSS scroll-driven animations run on the compositor thread, giving you buttery-smooth 60fps+ animations with zero main-thread jank.

3. scroll-timeline Property

Declares a named scroll progress timeline on a scroll container. Child elements (or the element itself) can reference it via animation-timeline.

Property Values Default
scroll-timeline-name none | <dashed-ident> none
scroll-timeline-axis block | inline | x | y block
scroll-timeline Shorthand: <name> <axis>
CSS/* Name a scroll timeline on a container */ .scroll-container { overflow-y: scroll; scroll-timeline-name: --page-scroll; scroll-timeline-axis: block; /* default */ } /* Shorthand */ .scroll-container { overflow-y: scroll; scroll-timeline: --page-scroll block; }

4. view-timeline Property

Declares a view progress timeline that measures how much of an element is currently visible inside a scrollport.

Property Values Default
view-timeline-name none | <dashed-ident> none
view-timeline-axis block | inline | x | y block
view-timeline-inset auto | <length-percentage>{1,2} auto
view-timeline Shorthand: <name> <axis>
CSS/* Track visibility of an element */ .card { view-timeline-name: --card-reveal; view-timeline-axis: block; } /* Shorthand */ .card { view-timeline: --card-reveal block; }

5. animation-timeline Core

Links an animation to a scroll timeline instead of the default document timeline (time-based). This is the single most important property for scroll-driven animations.

Value Description
auto Default document timeline (time-based)
none No timeline, animation won't progress
scroll() Anonymous scroll progress timeline
view() Anonymous view progress timeline
<dashed-ident> Named timeline (from scroll-timeline-name or view-timeline-name)
CSS@keyframes fade-in { from { opacity: 0; transform: translateY(40px); } to { opacity: 1; transform: translateY(0); } } .element { animation: fade-in linear both; animation-timeline: scroll(); /* tied to nearest scroll ancestor */ }
ℹ️ Note

When using animation-timeline, the animation-duration is ignored because progress is measured by scroll position, not time. Use auto or any value — it doesn't matter.

6. animation-range Core

Controls which portion of the timeline the animation maps to. Lets you fine-tune exactly when the animation starts and ends within scroll or view progress.

Property Values
animation-range-start normal | <length-percentage> | <timeline-range-name> <length-percentage>
animation-range-end normal | <length-percentage> | <timeline-range-name> <length-percentage>
animation-range Shorthand: <start> <end>

Timeline Range Names (for ViewTimeline)

cover
Full range: element enters → fully exits scrollport
contain
Element is fully visible inside scrollport
entry
Element starts entering → fully entered
exit
Element starts exiting → fully exited
entry-crossing
Element crosses the entry edge of scrollport
exit-crossing
Element crosses the exit edge of scrollport
CSS/* Animate only during the entry phase */ .card { animation: slide-up linear both; animation-timeline: view(); animation-range: entry 0% entry 100%; } /* Animate from 25% to 75% of scroll */ .progress { animation: grow linear both; animation-timeline: scroll(); animation-range: 25% 75%; } /* Cover range — full visibility cycle */ .element { animation: fade linear both; animation-timeline: view(); animation-range: cover 0% cover 100%; } /* Contain — only while fully in view */ .element { animation: scale-up linear both; animation-timeline: view(); animation-range: contain 0% contain 100%; }
💡 Tip

Use Chrome DevTools → Animations panel → "Scroll-driven" to visually debug animation-range values.

7. scroll() Function Anonymous

Creates an anonymous scroll progress timeline inline. No need to name a timeline.

Syntaxanimation-timeline: scroll(<scroller> <axis>); /* <scroller> values */ nearest /* default – nearest ancestor scroll container */ root /* document viewport / root scroller */ self /* the element itself (if it scrolls) */ /* <axis> values */ block /* default – block axis (vertical in LTR) */ inline /* inline axis (horizontal in LTR) */ x /* horizontal axis */ y /* vertical axis */
CSS/* Progress bar tied to root scroll */ .progress-bar { animation: grow-width linear both; animation-timeline: scroll(root block); } /* Element scrolls itself */ .self-scroll { overflow-y: scroll; animation: highlight linear both; animation-timeline: scroll(self); }

8. view() Function Anonymous

Creates an anonymous view progress timeline based on the element's visibility in its nearest scroll ancestor.

Syntaxanimation-timeline: view(<axis> <inset>); /* <axis> values */ block /* default */ inline x y /* <inset> – shrink the scrollport for trigger area */ auto /* default, no inset */ <length> /* e.g., 100px — both start and end */ <percentage> /* e.g., 20% */ <len> <len> /* start-inset end-inset */
CSS/* Fade in when element enters viewport */ .card { animation: fade-in linear both; animation-timeline: view(); } /* With inset — trigger 100px before reaching edge */ .card { animation: fade-in linear both; animation-timeline: view(block 100px); } /* Different start/end insets */ .card { animation: fade-in linear both; animation-timeline: view(block 100px 50px); }
ℹ️ Insets Explained

view(block 100px) means the scrollport is virtually shrunk by 100px on both sides. The element must scroll 100px past the edge before the animation begins. Great for "trigger before reaching screen edge" patterns.

9. Named Timelines Advanced

Named timelines let you explicitly reference a specific scroller or element, enabling cross-element animations.

CSS/* Container defines the named scroll timeline */ .scroll-container { overflow-y: scroll; scroll-timeline: --main-scroll block; } /* Any descendant can reference it */ .progress-bar { animation: grow linear both; animation-timeline: --main-scroll; } .sidebar-indicator { animation: move-dot linear both; animation-timeline: --main-scroll; } /* Named view timeline */ .hero-image { view-timeline: --hero-visibility block; } .hero-caption { animation: fade-in linear both; animation-timeline: --hero-visibility; animation-range: contain; }

10. timeline-scope Advanced

By default, a named timeline is only accessible to descendants. timeline-scope on an ancestor "hoists" the timeline's scope so sibling elements can reference it.

CSS/* The common ancestor hoists the scope */ .layout { timeline-scope: --card-timeline; } /* Child A: defines the timeline */ .layout .card { view-timeline: --card-timeline block; } /* Child B (sibling of .card): uses the timeline */ .layout .sidebar-marker { animation: slide linear both; animation-timeline: --card-timeline; }
⚠️ Important

timeline-scope must be set on a common ancestor of both the timeline source and the animation consumer. It doesn't create a timeline — it merely widens accessibility of an existing named timeline.

18. Combining with @keyframes Patterns

Multiple Animations, Different Timelines

CSS@keyframes scroll-fade { from { opacity: 0; } to { opacity: 1; } } @keyframes pulse { 0%, 100% { scale: 1; } 50% { scale: 1.05; } } .card { /* First: scroll-driven, Second: time-based */ animation: scroll-fade linear both, pulse 2s ease-in-out infinite; animation-timeline: view(), /* for scroll-fade */ auto; /* for pulse (regular time) */ }

Keyframes with Percentage Ranges

CSS@keyframes multi-step { 0% { opacity: 0; transform: scale(0.8); } 30% { opacity: 1; transform: scale(1); } 70% { opacity: 1; transform: scale(1); } 100% { opacity: 0; transform: scale(1.1); } } /* Element fades in at entry, stays visible, fades out at exit */ .element { animation: multi-step linear both; animation-timeline: view(); animation-range: cover; }

animation-fill-mode Behavior

Fill Mode Before Range Start After Range End
none No applied styles No applied styles
forwards No applied styles Final keyframe
backwards First keyframe No applied styles
both First keyframe Final keyframe
💡 Tip

For scroll-driven animations, always use animation-fill-mode: both so the element holds its animated state before entering and after exiting the animation range.

19. JavaScript Fallback Compat

For browsers without support, use the scroll-timeline polyfill or a lightweight IntersectionObserver fallback.

JS — Polyfill<!-- Add before your CSS to polyfill scroll-driven animations --> <script src="https://flackr.github.io/scroll-timeline/dist/scroll-timeline.js"></script>
JS — IntersectionObserver Fallback// Feature detection + fallback if (!CSS.supports('animation-timeline', 'scroll()')) { const observer = new IntersectionObserver((entries) => { entries.forEach(entry => { if (entry.isIntersecting) { entry.target.classList.add('visible'); } }); }, { threshold: 0.1 }); document.querySelectorAll('.reveal-card').forEach(el => { observer.observe(el); }); } // Scroll progress bar fallback if (!CSS.supports('animation-timeline', 'scroll()')) { const bar = document.querySelector('.progress-bar'); window.addEventListener('scroll', () => { const scrolled = window.scrollY; const height = document.body.scrollHeight - window.innerHeight; bar.style.transform = `scaleX(${scrolled / height})`; }, { passive: true }); }
CSS — Fallback Classes/* Styles for JS fallback */ @supports not (animation-timeline: scroll()) { .reveal-card { opacity: 0; transform: translateY(60px); transition: opacity 0.6s ease, transform 0.6s ease; } .reveal-card.visible { opacity: 1; transform: none; } }

20. Best Practices Guide

# Guideline Why
1 Use transform & opacity only Compositor-friendly — avoids layout/paint, smooth 60fps
2 Always add animation-fill-mode: both Prevents flashing before/after the animation range
3 Use @supports for feature detection Graceful degradation in unsupported browsers
4 Prefer view() for element-level animations Simpler than named timelines for single-element use
5 Prefer scroll(root) for page-level animations Explicit about which scroller drives the animation
6 Use will-change: transform sparingly Promotes to own layer — helpful for complex cases, wasteful for simple ones
7 Test with Chrome DevTools Animations panel Visual debugging of scroll-driven timelines and ranges
8 Use linear easing for scroll animations Non-linear easing feels odd because user controls speed via scrolling
9 Add polyfill for production sites Safari & Firefox support is still limited
10 Avoid animating height, width, top, left These trigger expensive layout recalculations
💡 Performance Note

CSS scroll-driven animations are significantly faster than JS scroll listeners because they run off the main thread. Prefer them over requestAnimationFrame scroll handlers whenever possible.
